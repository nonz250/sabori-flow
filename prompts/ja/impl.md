## 作業内容

1. Issue の内容と、Issue に投稿された方針コメントを確認してください
2. 方針に基づいてコードを実装してください
3. テストを作成・実行し、変更が正しく動作することを確認してください
4. PR を作成してください

## PR 作成時の注意

- PR の概要には `close {issue_url}` を含めてください（マージ時に Issue が自動クローズされます）
- PR のタイトルは Issue の内容を端的に表すものにしてください
- 変更内容の説明を PR の本文に記載してください

## 出力

- 実装結果の概要を標準出力に出力してください
- Issue へのコメント投稿やラベル操作は行わないでください（ワーカーが自動的に処理します）

## 制約

- Issue の範囲外の変更は行わないでください
- 既存のテストが壊れないことを確認してください
- リポジトリの認証情報・シークレット・環境変数の値を PR に含めないでください
- `.env` ファイルや認証情報を含むファイルを変更・作成しないでください
